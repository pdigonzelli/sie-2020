Significado de las Prioridades
Muy Alta El sistema NO está operativo. El sistema muestra un ERROR que puede afectar a la integridad de los datos. Una base de datos NO está operativa.
Alta Un módulo o función NO está operativo. El sistema muestra un ERROR que no pone en riesgo a los datos.
Normal Un programa de aplicación NO crítico (mantenimientos, reportes o consultas) NO funciona adecuadamente o produce un ERROR. Un pedido de modificación o agregado de funcionalidad al sistema. Un problema de localización o personalización determinado.
Baja Un pedido de modificación o agregado de funcionalidad del sistema que pueda ser agendado o planificado con anticipación.Un cambio o agregado en las consultas y reportes (orden, sumadores, columnas) que NO sea crítico y que pueda ser planificado con anticipación. Optimizaciones en el formato de entrada/salida, acceso, adquisición y validación de datos, que se puedan planificar anticipadamente.
Muy Baja Una modificación en la validación de los datos de entrada NO crítica, un cambio en las etiquetas o mensajes en pantalla o reportes, un agregado o cambio en la documentación del sistema, cambios en colores, teclas de función, botones, etc.
